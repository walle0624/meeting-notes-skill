# 会议纪要 (Meeting Notes)

把一次会议里零散的信息(口述、草稿、录音转写、聊天记录)整理成**结构统一**的会议纪要,并写入用户的 Obsidian 知识库。目标是:用户随口说"记一下今天的会"就能得到一份干净、可检索、可追踪待办的纪要。

## 保存位置

所有会议纪要写入这个目录:

```
/Users/walle/Obsidian/会议纪要/
```

若目录不存在就先创建(`mkdir -p`)。一次会议对应一个文件。

## 文件命名规则

格式固定为 **`会议名称-日期-地点.md`**,这样在文件列表里一眼能看出"什么会、什么时候、在哪"。

- **日期**用 `YYYY-MM-DD`。"今天/明天/昨天/上周三"等相对说法,按当前实际日期换算;用户没说日期就默认今天,并在最后顺带确认一句。
- **同日同地多场会**:在日期后加时间 `YYYY-MM-DD-HHmm`(如 `2026-06-24-1430`),避免撞名。
- **清理非法字符**:把 `/ \ : * ? " < > | # ^ [ ]` 和换行替换成 `-`(这些字符在文件系统或 Obsidian 双链里有特殊含义)。中文、空格、英文、数字、`-`、`_` 都可保留,Obsidian 完全支持。把连续的 `-` 合并成一个。
- **重名处理**:写入前先确认目标文件是否已存在。若存在且确属另一场会,就在末尾加时间或 `-2`、`-3`,**绝不直接覆盖**已有纪要。

示例:`产品评审会-2026-06-24-3楼会议室A.md`、`周会-2026-06-24-腾讯会议.md`

## 内容结构

严格按下面的模板产出。开头的 YAML frontmatter 是给 Obsidian 用的(属性面板 + Dataview 检索),正文是六个固定小节,正好覆盖:主题(标题)、参会人员/时间/地点、议题、过程、结论、待办。

```markdown
---
会议: <会议名称>
日期: <YYYY-MM-DD>
地点: <地点>
参会人员:
  - <姓名>
  - <姓名>
tags:
  - 会议纪要
---

# <会议名称>

## 会议信息
- **时间**:<YYYY-MM-DD HH:mm>
- **地点**:<地点>
- **参会人员**:<姓名、姓名……>

## 议题 Topic
- <本次会议要讨论的话题 / 目标>

## 过程记录
<讨论的展开:各方观点、关键数据、分歧点、决策依据。保留有信息量的内容,去掉口水话。可用小标题或列表按议题分段。>

## 结论
- <达成的结论 / 决议 / 拍板的方案>

## 待办事项
- [ ] <要做的事> —— 负责人:<谁> —— 截止:<日期>
- [ ] <要做的事> —— 负责人:<谁> —— 截止:<日期>
```

几点说明,理解了再动手会做得更好:

- **frontmatter 里的字段和正文"会议信息"会重复一部分**(日期、地点、参会人员),这是有意为之:frontmatter 给机器检索(以后能用 Dataview 列出"某人参加过的所有会""本月所有会议"),正文给人阅读。两边保持一致即可。
- **待办用 `- [ ]` 复选框**,因为 Obsidian 能聚合全库的未完成任务,这样会后行动项不会丢。尽量写清"做什么 + 谁 + 何时";信息不全也先记下来,把缺的部分标成 `待定` 而不是省略。
- **过程记录**是纪要的主体,不要压缩成一句话;但也不必逐字转录,提炼成"谁主张什么、为什么、最后怎么定"。

## 工作流程

1. **提取信息**:从用户给的内容(对话描述、粘贴的草稿、或让你读取的转写文件)里抽出:会议名称、日期、时间、地点、参会人员、议题、过程要点、结论、待办。
2. **缺关键信息就一次问清**:会议名称、日期、地点、参会人员这几项关乎文件名和抬头,如果缺,**集中一次性追问**(别一条一条来回拉扯)。议题/过程/结论这类正文内容,用户给多少整理多少。
3. **整理成结构**:套用上面的模板,语言跟随用户(默认中文)。不要编造没发生的内容——没有的结论、没点到的人名就留空或标 `待补充`,这是纪要,真实性优先。
4. **若用户提供了原始文件**(录音转写 `.txt/.md`、草稿等),先读取再整理,而不是凭空猜。
5. **写入文件**:按命名规则定好文件名,新建文件写到 `/Users/walle/Obsidian/会议纪要/`。写之前确认不会覆盖同名文件。
6. **反馈用户**:给出最终文件的完整路径,并指出哪些字段还空着需要他补(比如待办没定截止日期)。

## 一个完整示例

**用户说**:"帮我记下今天的产品评审会,在3楼会议室,我、老李、产品组小王参加。主要评审 v2 改版方案。讨论下来首页要重做,老李担心工期。最后定了:首页重做,小王下周三前出原型。"

**应产出文件** `/Users/walle/Obsidian/会议纪要/产品评审会-2026-06-24-3楼会议室.md`:

```markdown
---
会议: 产品评审会
日期: 2026-06-24
地点: 3楼会议室
参会人员:
  - 我
  - 老李
  - 小王
tags:
  - 会议纪要
---

# 产品评审会

## 会议信息
- **时间**:2026-06-24
- **地点**:3楼会议室
- **参会人员**:我、老李、小王(产品组)

## 议题 Topic
- 评审 v2 改版方案

## 过程记录
- 评审 v2 改版方案,焦点集中在首页。
- 一致认为现有首页需要重做。
- 老李提出对工期的担忧,需要评估重做对排期的影响。

## 结论
- 首页确定重做。

## 待办事项
- [ ] 产出首页重做原型 —— 负责人:小王 —— 截止:下周三(2026-07-01)
- [ ] 评估首页重做对整体工期的影响 —— 负责人:老李 —— 截止:待定
```

写完后告诉用户文件路径,并提示:"老李那项的截止日期还没定,要补吗?"
